# General Contract: work-dir Abstraction

This file is referenced by the 3rd-review SKILL.md thin shell. The main session does not read it; adapter entry points / sub-agents read it on demand.

The review strategy layer (SKILL.md references/ + verdict scripts) only recognises the abstract **work-dir** (the on-disk root for review output) and does not hard-code any platform-specific details:

**agenthub adapter**: work-dir is the task directory (`--task-dir`; missing value triggers fail-fast with a prompt indicating `task-dir` is required). Review identity comes from the `checkpoint_request` `reviewRequestId`.
**standalone adapter**: work-dir is `<output-root>/tasks/<name>/` (task name generated by O9). Review identity comes from a request id self-generated by standalone.

Adapter differences live exclusively in `review-dispatch-adapter.sh` (agenthub platform path; not in the standalone repo) and `./standalone.sh` (off-platform); the rest of this skill treats both identically.
